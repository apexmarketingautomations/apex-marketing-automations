Build a standalone digital business card product with a built-in sales referral system.

PROJECT GOAL
I want a public-facing digital business card product that lets business owners create their own card without needing a subscription just to access it. This must be a simple one-time purchase flow for now. After someone buys, they get their digital business card and also get their own referral link/code so they can earn commission when other people buy through them.

IMPORTANT BUSINESS RULES
- This is NOT the full Apex platform.
- This is a standalone product flow.
- No subscription is required to access the public page.
- No login is required before creating or previewing a card.
- The user should be able to enter their info, preview their card, and purchase it.
- After purchase, they can access their card and referral info.
- The referral system should only reward actual completed purchases.
- This should be lightweight, mobile-first, and launchable fast.

CORE PRODUCT FLOW
1. Visitor lands on a public standalone sales page.
2. Visitor clicks to create their digital business card.
3. Visitor fills out a form with their business information.
4. The system generates a live preview of their digital card.
5. Visitor completes a one-time Stripe payment to publish/activate the card.
6. After successful payment:
   - create the customer record if needed
   - create the card record
   - generate a unique referral code for that buyer
   - show them their live card URL
   - show them their referral link
7. Every published customer card should include a CTA button near the bottom:
   - "Get Your Own Card"
   - that button should automatically include the card owner’s referral code in the URL
8. If a new buyer purchases through that referral link, the original card owner gets referral credit.
9. Referral commission should be tracked in the database and shown to the referring user.
10. Payouts should be manual for now, not automated.

TECH REQUIREMENTS
- Use Replit’s built-in SQL database
- Use environment variables / Replit Secrets for all sensitive keys
- Use Stripe for one-time payment processing
- Use Stripe webhooks to confirm successful payments before marking an order as paid
- Build clean mobile-first responsive UI
- Make the code easy to edit later
- Prepare the app for deployment on Replit

REFERRAL SYSTEM LOGIC
I want a simple referral system tied only to actual paid purchases.

Referral logic requirements:
1. Every paid buyer gets a unique referral code.
2. Their card automatically includes a "Get Your Own Card" button using their referral code.
3. Referral link format should be something like:
   /card?ref=CODE123
   or
   /create?ref=CODE123
4. When a visitor lands through a referral link:
   - store the ref code in session/cookie
   - preserve it through preview and checkout
5. If that visitor completes a successful Stripe payment:
   - create an order record
   - create a referral record
   - assign the commission to the referring user
   - set the commission status to "pending"
6. Do not create commission records for unpaid, failed, or abandoned checkout attempts.
7. Prevent self-referrals:
   - if buyer email matches referrer email, do not count it
8. Only count one commission per paid order.
9. Manual payout workflow only for now.
10. Commission amount should be a flat fixed amount.
