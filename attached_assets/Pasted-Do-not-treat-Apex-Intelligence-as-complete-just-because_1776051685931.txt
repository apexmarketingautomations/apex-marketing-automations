## PART 1 — UI / UX HARDENING

### 1. Audit current UI vs backend capabilities
For every backend feature already built, verify:

- visible surface
- status indicator
- explanation layer
- management control
- action path
- proof/testing visibility

If missing → must be built.

### 2. Required Apex Intelligence UI surfaces

#### A. Executive Command Layer
- overall platform health
- launch readiness
- top risks
- top opportunities
- system health summary
- module coverage summary
- unresolved blockers
- priority action queue

#### B. Live Intelligence
- live event stream
- filtered timeline
- entity activity timelines
- module-by-module activity
- account / sub-account intelligence state

#### C. Scores
- lead intent
- launch readiness

#### D. Recommendations
- active recommendations
- priority

#### E. Health + Trust Layer
- integration health
- failure visibility

## PART 2 — PRODUCTION SEED + LIVE INITIALIZATION

Apex Intelligence must NOT go live as “empty infrastructure”.

### REQUIRED PRODUCTION BASELINE

The production environment must include:

- module event registry populated
- module coverage baseline records
- intelligence score definitions
- recommendation type definitions
- integration health baseline states
- execution timeline baseline markers
- required config/enums/state records
- UI-required baseline records (if needed)
- readiness baseline data

---

### PRODUCTION SEED RULES

- idempotent (safe to run multiple times)
- non-destructive
- additive only
- environment-aware
- no demo/test data in production

---

### SEPARATION

- dev seed ≠ production seed
- production seed = clean baseline only

---

### PUBLISH REQUIREMENTS

On deploy/publish:

System must verify:
- tables exist
- indexes exist
- required baseline records exist
- workers won’t start empty
- UI will not render blank/broken

If missing:
- block publish OR
- raise critical warning

---

### IMPLEMENTATION REQUIREMENTS

You must:

- create production seed script/process
- define required baseline records explicitly
- ensure safe upsert behavior
- log all seed actions
- verify seed success
- expose readiness state in admin/operator UI

---

### HARD RULE

Do NOT ship:
- empty intelligence tables
- unseeded registry
- missing config
- backend-only completion

Production must be:
👉 initialized  
👉 usable  
👉 trustworthy
