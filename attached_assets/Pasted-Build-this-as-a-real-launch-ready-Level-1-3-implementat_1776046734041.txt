LEVEL STRUCTURE

LEVEL 1 — FULL SITEWIDE DATA CAPTURE + APEX BRAIN FOUNDATION

Build the complete event, identity, and reporting foundation.

Requirements:
- Create a universal event tracking architecture for all sites/pages/components built through the Liquid website builder and Apex site properties.
- Instrument all core interactions automatically, not manually page by page.
- Create a shared event schema so all captured actions are standardized.
- Build identity stitching logic across anonymous visitor, known lead, returning contact, and customer states.
- Capture source attribution, campaign data, landing page context, device/session data, and page journey data.
- Send all tracked data into Apex Intelligence in real time or near-real time.
- Create account-aware routing so events are linked to the correct sub-account/site while still supporting ecosystem-wide intelligence.
- Add event reliability protections: retry handling, deduplication, fallback queueing, malformed payload handling, and status/error logging.
- Create a visible admin/operator UI showing:
  - data capture status
  - live event stream
  - failed events
  - account routing status
  - tracking health
  - source breakdown
  - top pages
  - top CTAs
  - top forms
  - event volume by site/sub-account
- Add settings UI for:
  - which event families are enabled
  - which sites are connected
  - which builder components are tracked
  - field mapping rules
  - consent/privacy handling controls where relevant
- Make the system feel like a premium AI command center, not a raw logs page.

LEVEL 2 — APEX INTELLIGENCE ACTION LAYER FOR OPTIMIZATION + ADAPTATION

Now make Apex Intelligence actually use the data.

Requirements:
- Build intelligence services that interpret captured site behavior and generate actionable outputs for each sub-account and for Apex globally.
- Use the data to score visitor intent, lead quality, funnel strength, engagement quality, and conversion readiness.
- Generate recommendations for:
  - better CTAs
  - better form structure
  - better page sequencing
  - better section ordering
  - better content emphasis
  - better follow-up timing
  - better segmentation
  - better offer matching
- Enable Apex Intelligence to support dynamic adaptation logic such as:
  - recommend alternate page sections
  - recommend shorter or smarter forms
  - recommend stronger agent responses
  - recommend better booking prompts
  - recommend better nurture paths
  - identify drop-off points and likely causes
- Build operator-facing UI for:
  - intelligence summaries per site
  - opportunity alerts
  - friction alerts
  - conversion blockers
  - top-performing patterns across the network
  - recommended fixes by priority
  - lead intent scoring views
  - journey analysis views
- Build sub-account views that make the intelligence usable by operators, not buried in technical dashboards.
- Add clear “why this recommendation exists” logic so the system is interpretable and trusted.
- Support both per-account intelligence and cross-account pattern learning where appropriate.
- Make sure Apex can learn from aggregate performance without contaminating or breaking tenant/account boundaries.

Turn Apex Intelligence into the operating brain across the full website stack.
