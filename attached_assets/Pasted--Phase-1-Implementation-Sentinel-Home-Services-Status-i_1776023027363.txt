## File 1: `shared/schema.ts`

Two surgical additions. No structural changes.

**Addition 1 — `sentinelConfig` table definition:**

Add `niche` as the second-to-last field, before `updatedAt`. Order matters for readability and convention.

```typescript
export const sentinelConfig = pgTable("sentinel_config", {
  id: serial("id").primaryKey(),
  subAccountId: integer("sub_account_id").references(() => subAccounts.id).notNull(),
  feedUrl: text("feed_url"),
  keywords: text("keywords").array().default([]),
  scanInterval: integer("scan_interval").default(60),
  enabled: boolean("enabled").default(false),
  smsAlertEnabled: boolean("sms_alert_enabled").default(true),
  geofenceEnabled: boolean("geofence_enabled").default(true),
  geofenceRadiusMiles: real("geofence_radius_miles").default(1),
  smsAlertPhone: text("sms_alert_phone"),
  targetCities: text("target_cities").array().default([]),
  targetStates: text("target_states").array().default([]),
  niche: text("niche").notNull().default("accident"),   // ← ADD THIS LINE ONLY
  updatedAt: timestamp("updated_at").defaultNow().notNull(),
});
```

`insertSentinelConfigSchema` is auto-generated by `createInsertSchema(sentinelConfig).omit({ id: true, updatedAt: true })`. Because `niche` has a DB-level default, drizzle-zod will make it optional in the generated schema automatically. `SentinelConfig` type will include `niche: string` automatically. No manual schema override needed.

**Verify:** After saving, TypeScript should compile. `SentinelConfig.niche` should be `string`. `InsertSentinelConfig` should allow `niche` as an optional field.

## Production Risks — Flag Before Deploying

**Risk 1:** If the application boots before the migration runs, existing code will not crash (the column doesn't exist yet, but no existing query selects `niche` until this deploy). Deploy order should be: migration first, application second. Verify migration completed before rolling the app.

**Risk 2:** The `(config as any).niche` cast in the frontend is intentional because `SentinelConfig` type from the shared schema may lag if the TypeScript build cache is stale. After rebuilding shared types, replace `(config as any).niche` with `config.niche` everywhere in the frontend — the cast is a deploy-safety measure, not permanent.

**Risk 3:** If any other file in the codebase reads `SentinelConfig` shape and makes assumptions about its fields (e.g., spreads it into another object), verify those callsites compile after adding `niche`. The storage layer's `upsertSentinelConfig` spreads `data` directly — this is safe because `niche` is a valid column and will be included automatically.
