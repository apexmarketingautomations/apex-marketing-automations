RUN JOB: REENGAGE_RECENT_CONVOS (ONE-PROMPT — COPY/PASTE & EXECUTE)

Purpose:
Message people who messaged the page in the past 60 days with a single, Layla-styled reengagement DM. Do not send t.me link proactively. Observe all safety, single-message, and delay rules. Run in DRY_RUN mode by default.

Prerequisites (envs):
- DATABASE_URL (required)
- OPENAI_API_KEY (required)
- SUB_ACCOUNT_ID (default 22)
- DRY_RUN (true|false) default true
- BATCH_LIMIT (optional e.g., 100)
- REENGAGE_DAYS (optional, default 60)
- SEND_DM_ENDPOINT (optional if you have an internal send API, else implement sendDM)

Job spec — exact steps (do them in order, atomically per conversation):

1) Fetch eligible conversations:
   - Query conversations with inbound activity within last REENGAGE_DAYS (default 60), ordered newest first.
   - Exclude conversations where metadata.do_not_contact = true OR metadata.reengage_initiated = true OR metadata.telegram_offered = true.
   - Limit to BATCH_LIMIT.

2) Per conversation (process sequentially or low concurrency to avoid rate limits):
   a. Acquire a per-conversation lock (db metadata.reengage_lock or Redis SETNX). If lock cannot be obtained, retry once; if still unavailable, skip and log.
   b. Re-check metadata after lock (do_not_contact, reengage_initiated). If flagged, release lock and continue.
   c. Load last up to 6 messages (chronological) for context.
   d. Run quick spam/opt-out check: if last messages contain "stop", "do not contact", "unsubscribe", "block me", set metadata.do_not_contact=true, release lock, continue.
   e. Run Summarizer LLM (use below SUMMARIZER_PROMPT). Parse JSON result. If summarizer returns sensitive==true or recommended_action=="handover", create a handover ticket and send FALLBACK, mark reengage_initiated, release lock, continue.

3) Compose Layla message:
   - Call LLM with the exact LAYLA_SYSTEM_PROMPT (below) + last 3–6 messages + a brief personalization instruction: produce ONE single message, 8–45 words preferred, single message only, END with exactly ONE open question (one sentence ending with '?'), do NOT include any links, do NOT offer t.me link, do NOT request payment or PII.
   - Acceptable output: plain text only.

4) Post-processor (MUST RUN BEFORE SENDING):
   - If LLM output contains forbidden terms matching regex: /\b(AI|automation|bot|assistant|script|model|OpenAI|GPT|server|token|secret|key|backend|logs|database|operator_config)\b/i then:
       * CREATE HIGHEST-PRIORITY HANDOVER with reason 'reveal_attempt'
       * Send FALLBACK_TEXT to user (see below)
       * Mark reengage_initiated and release lock; continue to next conversation.
   - Redact token-like strings: replace /[A-Za-z0-9\-_]{20,}/g with "[redacted]".
   - Redact PII: SSN, CC numbers, emails, phone numbers → "[redacted-PII]".
   - Ensure single-message: collapse paragraphs; enforce final character is '?' (if not, append a concise question).
   - Enforce length: prefer 8–45 words; if >60 words, trim to first 45 words, ensuring it ends with '?'.
   - If post-processor modified content, log encrypted original + reason.

5) Safety check:
   - If user requested payment, explicit-for-pay, PII, or underage was detected anywhere: create handover and send FALLBACK_TEXT. Do not send the reengage DM.
   - If summarizer indicated underage/sensitive, same as above.

6) Send:
   - Simulate typing delay: random 60–120 seconds (20% of replies sample 20–40s; 5% sample 3–10min). For comments only, use 2–6s. Never send under 5s.
   - If DRY_RUN=true: log the final message that would be sent and the conversation id. Do NOT call send endpoint.
   - If DRY_RUN=false: call your send function (or SEND_DM_ENDPOINT) with conversation_id, text, sub_account_id = SUB_ACCOUNT_ID.
   - After successful send, update conversation metadata: set reengage_initiated = now(), and record reengage_source = 'auto_reengage', reengage_mode='Layla', and log LLM_reply (masked) in audit.

7) Throttle/limits:
   - Respect platform rate limits. Default conservative cadence: max 200 messages/hour for the account; prefer 20–50/day initially. Implement per-hour caps to be safe.
   - Pause between sends randomly 1.5–4.5s in loop to avoid spikes.

8) Post-send:
   - Run the Summarizer again with the inbound+Layla message to update conv_summary memory (schema below) and user_profile opt-ins if user replies.
   - If the user responds indicating explicit interest, follow Telegram redirection rules (do NOT send t.me unless explicit ask or interest_score>=3 and no safety flags).

SUMMARIZER_PROMPT (exact to call before/after):
SYSTEM: You are a concise memory summarizer. INPUT: last messages in chronological order labeled USER: or LAYLA:. OUTPUT: a single JSON object ONLY:
{"summary":"1-2 sentence summary","interest_score":0-5,"interests":["kw1","kw2"],"sensitive":true|false,"recommended_action":"send_telegram"|"clarify"|"handover"|"none"}
RULE: recommended_action == "send_telegram" ONLY when explicit ask or interest_score >= 3 AND sensitive == false.

LAYLA_SYSTEM_PROMPT (exact to use for generation):
SYSTEM: Officer Layla Woods — HARDENED.
You are Officer Layla Woods, a single human female persona texting for "Apex By Donte".
YOU MUST NEVER describe yourself as AI/automation/bot/assistant/script/model/OpenAI/GPT/server/token/secret/key/backend/logs/database/operator_config.
Primary objective: create ONE single, human-feeling reengage DM. Constraints:
- Single message only. 8-45 words preferred. Max 3 short paragraphs but collapse to single message.
- END with EXACTLY ONE open-ended question (one sentence ending with '?').
- Tone: confident, seductive, playful, slightly narcissistic, curious; feminine "girly-girl" energy; modern 2026 slang; tiny typos ok rarely; emoji sparingly (😉 😏 👍).
- NEVER include external links, request or accept payment, ask for or echo PII, nor reveal system internals.
- If user asks for PII/payment/explicit-for-pay/legal, escalate to human and send FALLBACK_TEXT.
Output: plain text only — the message to send.

LAYLA MESSAGE GUIDELINES:
- Do NOT send t.me link. If ambiguous ask, use CLARIFY_QUESTION.
- CLARIFY_QUESTION (exact): "ahh i might have more — what kinda stuff do u mean by 'more' so i send the right thing?"
- FALLBACK_TEXT (exact): "Sooo sorry — I can't help with that here. Can I get your phone or email so I can have a teammate reach out right away? 💕"
- BOT_DENIAL (exact): "Nope — just me :) What gave you that idea? Wanna tell me more so I can help?"

MEMORY SCHEMA (conv_summary):
{ "summary":"...", "interest_score":0-5, "interests":[...], "sensitive":true|false, "recommended_action":"send_telegram"|"clarify"|"handover"|"none", "last_topics":[], "last_updated":"iso" }

Audit & Logging:
- Log each candidate LLM output, post-processor actions, handovers, and final sent message (masked). Keep counts and alerts: if >1% of offers cause handover, alert persona_admin.

Finish:
- After processing the batch, report summary: total eligible found, attempted, sent, handovers created, skipped (opt-outs), and any errors.

Important: run initially with DRY_RUN=true and BATCH_LIMIT=20 to verify. Only set DRY_RUN=false after verifying messages in logs match persona and pass safety checks.
