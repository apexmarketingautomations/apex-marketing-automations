### 🔱 The Snapshot Marketplace: "B2B2B" Architecture

This is where your power users (Agencies) build a perfect setup for a niche (like "Exotic Car Rentals") and sell that "Blueprint" to other users. You provide the infrastructure; they provide the expertise.

#### 1. The "Snapshot" Schema

You need a way to "package" an entire account's DNA—workflows, bot prompts, site templates, and pipeline stages—into a single, shareable ID.

```prisma
// Add to schema.prisma

model Snapshot {
  id           String   @id @default(cuid())
  creatorId    String
  creator      User     @relation(fields: [creatorId], references: [id])
  name         String   // e.g., "The Ultimate Tattoo Studio Pack"
  description  String
  price        Float    @default(0.0) // Can be free or paid
  industry     String
  
  // The "DNA" Payload (stored as JSON)
  config       Json     
  
  isPublic     Boolean  @default(false)
  downloads    Int      @default(0)
  createdAt    DateTime @default(now())
}

```

#### 2. The "Publish" Logic

When an AGM or a GM builds a winning system for their bar (like your Toast integration + email capture), they can "God Mode" it into a product.

```javascript
// lib/marketplace-engine.ts

export async function publishSnapshot(subAccountId, details) {
  // 1. Capture the current state of the sub-account
  const accountData = await prisma.subAccount.findUnique({
    where: { id: subAccountId },
    include: { workflows: true, siteBuilder: true, voiceAgent: true }
  });

  // 2. Package it into a Snapshot
  const newSnapshot = await prisma.snapshot.create({
    data: {
      creatorId: accountData.ownerId,
      name: details.name,
      price: details.price,
      industry: accountData.industry,
      config: {
        vibe: accountData.vibeTheme,
        pipelines: accountData.pipelines,
        botPrompt: accountData.voiceAgent.prompt,
        workflowSteps: accountData.workflows.map(w => w.steps)
      }
    }
  });

  return newSnapshot.id; // This ID is now a viral link
}

```
