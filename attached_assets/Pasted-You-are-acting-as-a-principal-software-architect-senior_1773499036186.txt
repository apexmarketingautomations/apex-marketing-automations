PROJECT GOAL

Apex must evolve from a feature-heavy automation platform into a true AI-native business operating system.

That means:
- modules must no longer be tightly coupled through direct calls everywhere
- system actions must be driven through a central event architecture
- workflows, CRM, messaging, Sentinel, forms, ads, voice, analytics, and integrations must be able to react to shared events
- the platform must support a future Apex Operator agent that can observe system state, detect failures/opportunities, and take safe actions
- users should eventually be able to command the system in natural language while the platform executes safely in the background

PRIMARY OBJECTIVE

Design and begin implementing an incremental migration toward a centralized event-driven core.

This must be done safely, with minimal disruption, and in a way that supports:
- always-on background execution while users are offline
- auditability
- retry safety
- observability
- future AI operator integration
- maintainable scaling

PHASE 2 — TARGET ARCHITECTURE

Create a central Event Bus architecture with the following capabilities:

1. Standard event publishing
2. Standard event subscription
3. Event validation
4. Event deduplication
5. Event persistence / logging
6. Worker-safe asynchronous dispatch
7. Retry/error handling
8. Loop prevention / recursion guard
9. Per-module subscriber registration
10. Audit visibility

Use a clean event envelope such as:

{
  event_id: string,
  event_type: string,
  source_module: string,
  tenant_id?: string,
  entity_id?: string,
  timestamp: ISO string,
  payload: object,
  metadata?: object,
  correlation_id?: string,
  causation_id?: string,
  version?: number
}
