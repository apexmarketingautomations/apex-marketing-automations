I need to integrate a new module called roomOS into this existing Apex Marketing Automations codebase. roomOS is a Chaturbate performer tool — it receives live room events (tips, viewer enters, broadcast start/end) and powers a real-time mobile companion app with AI coaching, whale tracking, and session analytics.

Here is exactly what needs to be built:

## 2. NEW SERVER ROUTE — server/routes/chaturbate.ts

Create this file and register it in server/routes.ts as:
app.use('/api/chaturbate', chaturbateRouter)

Endpoints needed:

POST /api/chaturbate/webhook
- Receives events from the Chaturbate overlay (tip, enter, broadcast_start, broadcast_end)
- Verifies x-roomos-token header matches ROOMOS_WEBHOOK_SECRET env variable
- Finds sub_account by cb_username field
- On tip: updates in-memory session state, upserts contact in existing contacts table (source='chaturbate', external_id='cb_USERNAME'), emits to SSE channel roomos:{subAccountId}
- On enter: checks if entering user is a known contact with totalValue >= 100, if so emits whale_alert to SSE channel
- On broadcast_start: initializes fresh session state in memory Map, emits to SSE
- On broadcast_end: saves session to cb_sessions table, emits to SSE
- Always respond res.sendStatus(200) immediately before processing

GET /api/chaturbate/stream?subAccountId=X
- SSE endpoint using the existing SSE pattern from the unified inbox
- Subscribes to roomos:{subAccountId} channel
- Sends current session state immediately on connect

GET /api/chaturbate/sessions/:subAccountId
- Returns last 30 sessions from cb_sessions table ordered by session_date desc

GET /api/chaturbate/whales/:subAccountId
- Returns top 20 contacts where source='chaturbate' ordered by total_value desc

POST /api/chaturbate/command
- Logs a fired command to cb_commands_fired table
- Sets tokens_after and was_effective after 60 seconds using setTimeout

## 3. NEW AI SERVICES — server/services/roomOS/

Create server/services/roomOS/contextBuilder.ts
- Exports buildRoomContext(session, event, goalTokens) function
- Calculates room energy: hot (<30s since tip), warm (<90s), cooling (<180s), dead (180s+)
- Returns structured RoomContext object with all session stats

Create server/services/roomOS/aiCoach.ts
- Exports getRoomCoachingSuggestion(context, account) async function
- Uses the EXISTING aiGateway (OpenAI primary, Gemini fallback) — same pattern as the Layla persona
- System prompt: real-time room coach for Chaturbate performer, suggest single best chat message based on room state, under 12 words, natural tone
- Falls back to hardcoded message banks if AI fails
- Returns null if room energy is 'hot' and recent trigger was a tip (no suggestion needed)

## IMPORTANT NOTES

- The existing AI gateway already handles OpenAI + Gemini fallback with circuit breaker — use it exactly as the Layla persona uses it, do not create a new AI client
- The existing SSE infrastructure from the unified inbox handles real-time push — use the same sseManager pattern, do not create new SSE infrastructure
- The existing contacts table already handles multi-tenant CRM — upsert tippers into it using source='chaturbate' and external_id='cb_USERNAME' as the unique identifier
- Session state (in-memory Map) is intentionally not persisted to DB during a live session for performance — only save to cb_sessions on broadcast_end
- The CB app sends events via its overlay HTML (which runs in OBS browser source and CAN make fetch calls) — the overlay posts to POST /api/chaturbate/webhook with header x-roomos-token matching ROOMOS_WEBHOOK_SECRET

Do not change any existing functionality. This is purely additive.
