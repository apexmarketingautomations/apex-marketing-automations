This allows any user to take one of your core templates (like "Med Spa" or "Adult Creator") and "fork" it—making a copy they can refine, rename, and either use for a new client or re-publish to the marketplace under their own brand.

This is exactly how GitHub grew to own the world of code. You are doing the same for **Business DNA**.

### 🔱 The "Fork" Logic: Cloning the Business DNA

When a user "forks" a snapshot, you aren't just copying a row in a database; you are duplicating an entire ecosystem (Workflows, Agent Prompts, Site Structures) and re-linking them to a new owner.

```javascript
// lib/snapshot-fork-engine.ts

export async function forkSnapshot(snapshotId, newOwnerId, newBusinessName) {
  // 1. Fetch the Original "Parent" Snapshot
  const parentSnapshot = await prisma.snapshot.findUnique({
    where: { id: snapshotId }
  });

  if (!parentSnapshot) throw new Error("Snapshot not found.");

  // 2. CREATE THE FORKED SUB-ACCOUNT
  // We take the parent's 'config' (The DNA) and inject it into a new account
  const forkedAccount = await prisma.subAccount.create({
    data: {
      ownerId: newOwnerId,
      name: newBusinessName,
      industry: parentSnapshot.industry,
      vibeTheme: parentSnapshot.config.vibe,
      
      // The deep copy of the configuration
      config: parentSnapshot.config,
      
      // Track the lineage (Who is the original creator?)
      parentSnapshotId: parentSnapshot.id,
      isFork: true
    }
  });

  // 3. LOG THE FORK FOR THE MARKETPLACE (Social Proof)
  await prisma.snapshot.update({
    where: { id: snapshotId },
    data: { forkCount: { increment: 1 } }
  });

  return forkedAccount.id;
}

```
