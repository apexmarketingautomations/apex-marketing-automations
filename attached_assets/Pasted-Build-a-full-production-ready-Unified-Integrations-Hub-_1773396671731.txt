Build a full production-ready “Unified Integrations Hub” for Apex Marketing Automations from scratch.

Goal:
Replace the current developer-style integration system that asks users for OAuth client IDs and secrets with a simple user-friendly platform-managed OAuth system.

Core requirement:
Users should NEVER have to enter their own API keys, OAuth client IDs, or client secrets for standard integrations like Google, Meta, Gmail, Calendar, Sheets, Drive, Google Business Profile, Facebook, Instagram, and Meta Ads.

Instead, Apex must use platform-owned OAuth apps and one-click connect flows.

Build this with a clean architecture, secure token handling, and scalable integration management.

STACK:
- Frontend: React + TypeScript
- Backend: Node.js + Express or Next.js API routes
- Database: PostgreSQL with Prisma ORM
- Auth/session: secure server-side session handling
- OAuth library: use official OAuth 2.0 patterns
- UI: modern SaaS dashboard styling matching Apex brand
- Write all code from scratch, fully structured, production-grade

MAIN PRODUCT REQUIREMENTS:

1. INTEGRATIONS HUB UI
Create a new integrations page that groups services into simplified connection categories:

- Connect Google

- Connect Meta
  Covers:

- settings button

2. PLATFORM-MANAGED OAUTH
Implement platform-owned OAuth apps so that Apex stores the client IDs and secrets in environment variables and manages all auth flows centrally.

The user flow should be:

- user clicks “Connect Google”
- user is redirected to Google OAuth consent
- user approves scopes
- user is returned to Apex callback URL
- Apex securely stores tokens server-side
- UI updates to connected state

Do NOT ask the user to manually paste OAuth credentials.

- Gmail
