(If you’re not sure of your production domain, check Replit Deploy → Production → “Domain.” Use that instead of the placeholder.)

2) Test mode vs Live mode

Stripe webhooks are environment-specific, so you generally want two endpoints:

Test mode endpoint (when you’re using Stripe test keys)

Live mode endpoint (when you’re using Stripe live keys)

For launch readiness, you’ll create the Live one once you flip everything to live keys.

3) Events to listen for (simple + useful)

Start with the subscription-critical events:

checkout.session.completed

invoice.payment_succeeded

invoice.payment_failed (optional but helpful)

customer.subscription.updated

customer.subscription.deleted

That’s enough to keep subscriptions in sync (creates/renews/fails/cancels).
