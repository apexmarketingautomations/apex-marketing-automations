## Quick checklist to keep the agent honest

Always require this before approving any task:

* [ ] `SELECT * FROM <new_table> ORDER BY created_at DESC LIMIT 5;` output included
* [ ] Worker logs showing `worker-xxx started` and `job_id` timestamps
* [ ] Screenshots of new UI routes or component states
* [ ] Health endpoint output (`/api/health/<component>`)
* [ ] E2E proof: sequence of steps & SQL traces that prove the outcome
* [ ] If external platforms used, `integration_events` rows with redacted request/response
* [ ] If secrets were required but unavailable, `TEST_PUBLISH` evidence shown

If any of the above is missing — **reject and require rework**.
