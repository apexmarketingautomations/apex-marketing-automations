Extend the existing Apex tracking and intelligence system into a client-facing Apex Intelligence Dashboard.

DO NOT modify or break existing capture, attribution, identity stitching, or universal_events integration.

This phase is focused on:
- exposing intelligence to clients
- generating insights
- enabling real-time action
- maintaining Apex architectural integrity

SYSTEM CONTEXT:
- tracking_visits, tracking_events, card_intelligence_snapshots already exist
- identity stitching is implemented
- universal_events is the intelligence pipeline
- test traffic is already isolated
- attribution integrity is enforced

-------------------------------------

OBJECTIVE:
Turn internal tracking data into a visible, monetizable feature inside Apex.

-------------------------------------

PHASE 3 REQUIREMENTS:

1) ANALYTICS API EXTENSION

Create endpoint:
GET /api/intelligence/cards/:id

Return:

{
  cardId,
  metrics: {
    taps,
    qrScans,
    uniqueVisitors,
    repeatVisitors,
    identifiedVisitors,
    leads,
    bookedCalls,
    qualifiedLeads,
    conversionRate,
    repeatRate
  },
  behavior: {
    avgTimeToConvert,
    peakHours,
    topCTA,
    sessionDepth
  },
  attribution: {
    confidenceScore,
    stitchedJourneys,
    repeatClusters
  },
  recentActivity: [
    { type, timestamp, label }
  ],
  insights: []
}

Must:
- exclude test traffic
- use snapshot when available
- fallback to live aggregation if needed
- cache for 5 minutes (TTL)
- support ?refresh=1 to force recompute

-------------------------------------

2) INSIGHT ENGINE

Create service:
trackingInsights.ts

This must generate human-readable insights from metrics.

RULES (initial logic):

- if repeatRate > 0.25:
  "Repeat visitors are highly engaged and more likely to convert."

- if conversionRate < 0.1:
  "Your conversion rate is low. Consider optimizing your CTA or offer."

- if taps > 30 AND leads === 0:
  "Your card is getting attention but not converting. Review your landing page."

- if peakHours identified:
  "Most engagement happens around {peakHours}. Prioritize availability during this time."

- if identifiedVisitors increasing:
  "More visitors are being identified. Your follow-up system is working."

- if sessionDepth > threshold:
  "Visitors are exploring multiple sections. Interest level is high."

Return array:
[
  { type: "positive" | "warning" | "opportunity", message }
]

-------------------------------------

3) LIVE INTENT DETECTION

Add service:
trackingIntent.ts

Trigger conditions:

HIGH INTENT if:
- repeat visit within 24h
- AND (cta_click OR form_start)
- OR multiple sessions

When triggered:
- emit universal event:
  tracking.high_intent

- optional:
  call notification handler
